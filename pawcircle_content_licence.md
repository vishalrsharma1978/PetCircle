# eSamaj eBooks: Source + Format Migration Plan

## Goal

Fix the ebook where people can read sacred texts easily for free, while avoiding copyright or redistribution problems.

The site should not rely on random PDFs unless the source is clearly public domain, CC0, CC BY, or otherwise explicitly allows redistribution and commercial use.

For this project, "commercial use" includes:

* Hosting the texts for free on a public website
* Running the site as part of a business or portfolio
* Showing ads
* Using affiliate links (possibly)
* Generating EPUB/PDF downloads from hosted text
* Reformatting text into a custom reading experience

## Big Change

Do **not** build the main reading experience around downloaded PDFs.

Instead:

* Store texts as structured data or HTML
* Display texts in a clean web reader
* Make vertical scrolling the default
* Offer optional paginated/page mode later
* Offer PDF/EPUB downloads only when the source license allows it

## Main UX Decision

### Default reader mode

Use **scroll mode** as the default.

Reasons:

* Better on mobile
* Easier to search
* Easier to copy and quote
* Easier to resize text
* Easier to make accessible
* Better for SEO
* Better for long texts

### Optional reader mode

Add **page mode** as an optional toggle.

Page mode should be simple pagination, not a heavy animated page-flip effect.

Avoid making fake page-turning the default experience.

## File Format Decision

### Use for website reading

Use:

* HTML
* JSON
* Markdown
* EPUB-style structured text
* TEI/XML only if needed for scholarly structure

### Use for downloads only

Use:

* PDF
* EPUB

PDFs should be treated as downloads, facsimiles, or print versions — not the primary website reading format.

## Licensing Rules

Every text on the site must have a license status before publishing.

Allowed:

* Public domain
* CC0
* CC BY
* CC BY-SA, if share-alike requirements are followed
* Official source that explicitly allows redistribution and commercial use
* Publisher or translator permission in writing

Use with caution:

* Internet Archive scans
* Old PDFs from unknown uploaders
* PDFs that say "free download" but do not say redistribution is allowed
* Translations where the original text is public domain but the translation is copyrighted

Avoid:

* Non-commercial licenses
* "All rights reserved" PDFs
* Scans from unknown uploaders
* Modern translations without permission
* Sources that only allow personal, educational, or non-commercial use

## Important Licensing Notes

A text being ancient does **not** mean every modern edition or translation is free to reuse.

For each text, check separately:

1. Original source text
2. Translation
3. Commentary
4. Footnotes
5. Introduction
6. Formatting or editorial additions
7. Scans or images
8. Database structure

The original religious text may be public domain, while the translation or edition may still be copyrighted.

## Pali Tipitaka Change

### Old plan

Use PDF scans or Internet Archive copies of the Pali Tipitaka.

### New plan

Use **SuttaCentral / Bilara data** as the preferred source for Pali Buddhist texts.

Reason:

* SuttaCentral/Bilara provides structured text data.
* Supported translations are released under CC0/public-domain terms.
* CC0 is much safer for reuse, redistribution, reformatting, and commercial use.
* Structured text is better than PDF for web reading.

## Pali Tipitaka Sources

### Preferred source

SuttaCentral / Bilara data:

https://github.com/suttacentral/bilara-data

Use the `published` branch when possible.

### Source type

Use structured text data, not PDFs.

Expected formats:

* JSON
* HTML generated from JSON
* Static pages generated from source text
* Optional EPUB/PDF generated by our own site later

### Avoid as primary source

Do not use the VRI / tipitaka.org XML as the main commercial-safe source.

Reason:

* The VRI/tipitaka.org XML repository says the files are made available for non-commercial use.
* Non-commercial licensing is not safe for this website’s long-term plan.

Avoid:

https://github.com/VipassanaTech/tipitaka-xml

unless we receive written permission or keep it strictly outside commercial/reuse scope.

## Internet Archive Policy

Internet Archive can be useful for finding public-domain scans, but it should not be treated as automatically safe.

Use Internet Archive only when the specific item clearly says one of the following:

* Public domain
* CC0
* CC BY
* CC BY-SA
* No known copyright restrictions
* Clearly published before applicable copyright cutoff dates and verified

Do not assume every Internet Archive PDF is safe just because it is downloadable.

For each Internet Archive item, record:

* Title
* Author / translator
* Publisher
* Publication year
* Source URL
* Rights statement
* License status
* Whether commercial reuse is allowed
* Whether redistribution is allowed
* Notes

## Source Audit Table

Create a source audit table for every text.

Use this format:

| Text    | Tradition | Source              | Format | Original Text Status | Translation Status  | License           | Commercial Use? | Can Rehost? | Notes                    |
| ------- | --------- | ------------------- | ------ | -------------------- | ------------------- | ----------------- | --------------- | ----------- | ------------------------ |
| Example | Buddhist  | SuttaCentral/Bilara | JSON   | Public domain        | CC0 where supported | CC0/Public domain | Yes             | Yes         | Preferred for web reader |

Do not publish any text until this table is filled out.

Each text should have metadata.

```json

  "tradition": "Buddhist",
  "canon": "Pali Tipitaka",
  "collection": "Digha Nikaya",
  "language": "Pali",
  "source_name": "SuttaCentral / Bilara",
  "source_url": "https://github.com/suttacentral/bilara-data",
  "license": "CC0 / Public Domain where applicable",
  "commercial_use_allowed": true,
  "rehosting_allowed": true,
  "attribution_required": false,
  "attribution_recommended": true,

```

## Reader Features

Build the reader around text, not PDFs.

Minimum features:

* Table of contents
* Chapter/section navigation
* Search within text
* Global search across texts
* Font size control
* Line spacing control
* Light/dark mode
* Mobile-friendly layout
* Copy link to section
* Bookmark section
* Source/license info on every text page
* Download button only when allowed

Later features:

* Page mode toggle
* Side-by-side original and translation
* Notes/footnotes
* Audio
* Personal highlights
* Collections/playlists
* Offline EPUB generation

## Reader Page Layout

Each text page should include:

1. Text title
2. Tradition
3. Canon / collection
4. Translator or source
5. License notice
6. Reading controls
7. Table of contents
8. Main text
9. Source and attribution footer

Example footer:

```md
Source: SuttaCentral / Bilara  
License: CC0 / Public Domain where applicable  
Original Pali texts are treated as public domain.  
Translations and editorial material may have separate license terms.  
Please verify the license before reusing this text outside this website.
```

## Attribution Policy

Even when attribution is not legally required, include attribution when possible.

Recommended attribution format:

```md
Text data based on SuttaCentral / Bilara resources.  
SuttaCentral-supported translations are made available under CC0/public-domain terms where specified.  
Original Pali Buddhist texts are treated as public domain.
```

For non-CC0 texts, attribution must follow the license exactly.

## Download Policy

Do not offer downloads automatically.

Before adding a download button, confirm:

* The text can be redistributed
* Commercial use is allowed
* The translation can be reused
* The generated file does not include copyrighted notes, introductions, or formatting
* The license is displayed inside the downloaded file

Download types:

* `.html`
* `.epub`
* `.pdf`
* `.txt`
* `.json`

Recommended launch approach:

* Start with online reading only
* Add EPUB/PDF downloads later after license audit

## PDF Policy

PDFs are allowed only as:

* Public-domain scans
* Official PDFs with reuse permission
* PDFs generated by our own site from approved source text
* Facsimile downloads where license allows

PDFs should not be the main reading interface.

## EPUB Policy

EPUB is preferred for downloads because:

* It reflows on mobile
* It supports text resizing
* It is better for long reading
* It is easier to generate from structured text
* It is more reader-friendly than PDF

Generate EPUBs only from approved source text.

## Legal Safety Checklist

Before publishing any text, answer:

* Is the original text public domain?
* Is the translation public domain or openly licensed?
* Does the license allow commercial use?
* Does the license allow redistribution?
* Does the license allow format conversion?
* Does the license allow hosting on our own website?
* Are attribution requirements satisfied?
* Are share-alike requirements satisfied?
* Are non-commercial restrictions absent?
* Is the source URL saved?
* Is the license URL saved?
* Is the date checked saved?

If any answer is unclear, do not publish yet.

## Final Direction

The website should become a structured sacred-text reader, not a PDF library.

Use PDFs only when legally safe and useful as downloads.

The main product should be:

Clean searchable web text
